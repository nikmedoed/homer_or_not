# Homer or Not

Стартовая страница для Chrome/Chromium: локальный быстрый экран с поиском, быстрыми ссылками и плитками из Homer.

Главный принцип: страница работает сразу из расширения. Поиск и быстрые ссылки не зависят от Homer. Плитки, иконки сервисов и wallpaper подтягиваются из Homer после первой успешной синхронизации `assets/config.yml` на конкретном устройстве. Если Homer недоступен, расширение показывает локальный кеш, если этот браузер уже видел Homer. Новый браузер без кеша домашние плитки не показывает.

## Установка

1. Открой `chrome://extensions`.
2. Включи `Developer mode`.
3. Нажми `Load unpacked`.
4. Выбери папку `homer_or_not`.

Если папка лежит в git-репозитории, дальше достаточно делать `git pull` и нажимать `Reload` у расширения.

## Настройка

Базовые настройки лежат в `newtab.config.js`:

- `search.engines` — кнопки поиска и шаблоны URL с `{q}`;
- `search.defaultEngineId` — поисковик для Enter;
- `quickLinks` — верхняя строка быстрых ссылок: URL и необязательная подпись;
- `homer.url` — адрес Homer, от которого будет вычислен `assets/config.yml`.

Через кнопку `⚙` на странице можно менять поисковики, быстрые ссылки и URL Homer без редактирования файла. `Сбросить` возвращает состояние к `newtab.config.js`.

Для быстрых ссылок расширение само вытягивает favicon и title страницы, затем кеширует их локально. Если подпись у ссылки задана вручную, используется она. Если подписи нет, показывается обрезанный title страницы, а если его не удалось получить — домен.

Для поисковиков иконки тоже подтягиваются автоматически из favicon домена в `template`; вручную задавать букву, код, фон и цвет кнопки не нужно.

Настройки страницы хранятся в `chrome.storage.sync`, поэтому синхронизируются через аккаунт браузера. В `manifest.json` задан фиксированный `key`, чтобы unpacked-расширение получало один и тот же ID на разных машинах. Кеш Homer хранится в `chrome.storage.local`: новый браузер на рабочем компьютере не получит домашние плитки, пока сам не увидит Homer.

Homer обновляется при открытии новой вкладки, но не чаще одного раза в 5 минут. Если Homer задан локальным IPv4 и браузер отдает локальные IPv4-адреса через WebRTC, расширение сначала сверяет подсеть и не делает запрос, когда компьютер явно не в домашней локалке. Если браузер скрывает локальный IP, остается только экономичный пробный запрос с тем же 5-минутным ограничением.

## Homer

Поддерживаемые варианты URL:

- `http://192.168.1.28/`
- `http://192.168.1.28/assets`
- `http://192.168.1.28/assets/config.yml`

Иконки и wallpaper не входят в расширение. Они резолвятся относительно Homer-конфига, например `assets/icons/plex.svg` становится URL на стороне Homer.
